Je veux générer un dataset minimal pour tester les réponses JSON de mon projet TYPO3 v13 headless. 
Voici la stratégie à suivre :

1. Identifier les scénarios JSON à tester (ex: page simple, page avec contenu, page avec images, page avec catégories).
2. Pour chaque scénario, déterminer les tables nécessaires : pages, tt_content, sys_file/sys_file_reference (images), sys_category/sys_category_record_mm (catégories).
3. Extraire uniquement les lignes nécessaires de chaque table depuis la base de test.
4. Supprimer les champs dynamiques (crdate, tstamp, lastUpdated) et fixer des UID stables.
5. Exporter les données en CSV minimal par scénario.
6. Définir un JSON Schema par endpoint pour valider la structure et les types.
7. Générer un snapshot JSON à partir de ce dataset minimal.
8. Écrire un FunctionalTestCase TYPO3 qui :
   - importe le CSV
   - appelle le frontend via getFrontendResponse(uid)
   - valide le JSON contre le schema
   - compare le JSON avec le snapshot
9. Configurer le pipeline CI pour exécuter ces tests et bloquer la merge request si le test échoue.

Le résultat attendu est : dataset CSV minimal, JSON Schema, snapshot JSON, et FunctionalTestCase complet.
